## Overview

The Separator component provides a clean visual division between content sections. It's built on Radix UI's Separator primitive and supports both horizontal and vertical orientations with semantic meaning for screen readers.

## Features

- **Dual Orientation**: Horizontal and vertical separator support
- **Semantic HTML**: Proper ARIA roles for accessibility
- **Decorative Option**: Can be marked as decorative for screen readers
- **Flexible Styling**: Customizable appearance with CSS classes
- **Responsive Design**: Adapts to container dimensions
- **Accessibility**: Full screen reader support with proper semantics

## API Reference

### Separator

The main separator component built on Radix UI Separator primitive.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `orientation` | `"horizontal" \| "vertical"` | `"horizontal"` | The orientation of the separator |
| `decorative` | `boolean` | `true` | Whether the separator is decorative (no semantic meaning) |
| `className` | `string` | - | Additional CSS classes |

## Accessibility

The Separator component includes comprehensive accessibility features:

- **Semantic Roles**: Proper ARIA `separator` role when not decorative
- **Orientation Support**: Correct `aria-orientation` attribute
- **Decorative Option**: Can be marked as decorative to hide from screen readers
- **Keyboard Navigation**: Does not interfere with keyboard navigation flow

## Usage Guidelines

### When to Use

- **Content Sections**: Separate different content areas
- **List Items**: Divide items in lists or menus
- **Form Sections**: Group related form fields
- **Navigation**: Separate navigation items
- **Cards**: Organize content within cards

### When Not to Use

- **Spacing Only**: Use margin/padding instead for pure spacing
- **Complex Layouts**: Consider using CSS Grid or Flexbox
- **Interactive Elements**: Separators should not be interactive
