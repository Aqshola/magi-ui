# Scroll Area

Augments native scroll functionality for custom, cross-browser styling.

## Overview

The Scroll Area component provides a customizable scrolling container that replaces native browser scrollbars with styled alternatives. Built on Radix UI's ScrollArea primitive, it offers consistent cross-browser behavior and enhanced visual control.

## Features

- **Custom Styling**: Override default browser scrollbars with custom designs
- **Cross-browser**: Consistent appearance across different browsers  
- **Keyboard Navigation**: Full keyboard accessibility support
- **Touch Support**: Optimized for mobile and touch devices
- **Flexible Sizing**: Support for both vertical and horizontal scrolling
- **Performance**: Virtualization-ready for large datasets

### Controlled Scroll Position

<ComponentPreview name="scroll-area-controlled">

```tsx
import { ScrollArea } from "@/components/ui/scroll-area"
import { Button } from "@/components/ui/button"
import { useRef } from "react"

export function ControlledScrollDemo() {
  const scrollRef = useRef<HTMLDivElement>(null)

  const scrollToTop = () => {
    scrollRef.current?.scrollTo({ top: 0, behavior: 'smooth' })
  }

  const scrollToBottom = () => {
    scrollRef.current?.scrollTo({ 
      top: scrollRef.current.scrollHeight, 
      behavior: 'smooth' 
    })
  }

  return (
    <div className="space-y-4">
      <div className="flex gap-2">
        <Button onClick={scrollToTop} variant="outline" size="sm">
          Scroll to Top
        </Button>
        <Button onClick={scrollToBottom} variant="outline" size="sm">
          Scroll to Bottom
        </Button>
      </div>
      <ScrollArea className="h-[200px] w-full rounded-md border p-4" ref={scrollRef}>
        <div className="space-y-4">
          {Array.from({ length: 20 }, (_, i) => (
            <div key={i} className="p-4 border rounded">
              Content block {i + 1} - Lorem ipsum dolor sit amet, consectetur adipiscing elit.
            </div>
          ))}
        </div>
      </ScrollArea>
    </div>
  )
}
```

## Accessibility

The Scroll Area component follows WAI-ARIA guidelines:

- **Keyboard Navigation**: Supports arrow keys, Page Up/Down, Home/End
- **Screen Reader**: Properly announces scrollable regions
- **Focus Management**: Maintains focus within scrollable content
- **ARIA Labels**: Uses appropriate ARIA attributes for accessibility

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `↑` `↓` | Scroll vertically |
| `←` `→` | Scroll horizontally |
| `Page Up` `Page Down` | Scroll by page |
| `Home` `End` | Scroll to start/end |
| `Space` | Scroll down by page |

## Best Practices

### Content Organization
- Keep scroll areas focused on specific content types
- Use consistent heights for similar scroll areas
- Provide clear visual boundaries

### Performance
- Consider virtualization for large datasets (1000+ items)
- Implement lazy loading for dynamic content
- Use React.memo for scroll item components

### User Experience
- Show scroll indicators when content overflows
- Provide alternative navigation for long lists
- Ensure touch-friendly scrolling on mobile

### Styling
- Maintain consistent scrollbar appearance
- Use appropriate contrast for scrollbar visibility
- Consider dark mode scrollbar styling
