## Overview

The Context Menu component provides a way to display contextual actions when users right-click on elements. It's perfect for file managers, data tables, and any interface where secondary actions need to be easily accessible without cluttering the primary UI.

## Features

- **Right-Click Activation**: Triggered by right-click or long press on touch devices
- **Keyboard Navigation**: Full keyboard support with arrow keys and shortcuts
- **Nested Submenus**: Support for hierarchical menu structures
- **Checkbox & Radio Items**: Interactive menu items with state management
- **Customizable**: Easy to style and extend with additional functionality
- **Accessibility**: ARIA attributes and screen reader support
- **Portal Rendering**: Renders outside the DOM hierarchy to avoid z-index issues

## Best Practices

1. **Contextual Actions**: Only include actions relevant to the specific element
2. **Clear Labels**: Use descriptive, action-oriented labels for menu items
3. **Logical Grouping**: Group related actions together with separators
4. **Keyboard Shortcuts**: Include shortcuts for frequently used actions
5. **Destructive Actions**: Use the destructive variant for dangerous actions
6. **Submenu Depth**: Avoid deep nesting of submenus for better usability
