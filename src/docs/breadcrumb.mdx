## Overview

The Breadcrumb component provides a way to show users their current location within a site's hierarchy. It helps users understand where they are and provides an easy way to navigate back to parent pages.

## Features

- **Hierarchical Navigation**: Clear representation of page hierarchy
- **Customizable Separators**: Use different icons or text as separators
- **Dropdown Support**: Collapse long paths with dropdown menus
- **Icon Integration**: Support for icons alongside text
- **Accessibility**: Proper ARIA labels and semantic markup
- **Responsive**: Adapts to different screen sizes

## Examples

### Basic Implementation

```tsx
import {
  Breadcrumb,
  BreadcrumbItem,
  BreadcrumbLink,
  BreadcrumbList,
  BreadcrumbPage,
  BreadcrumbSeparator,
} from "@/components/ui/breadcrumb"

function BasicBreadcrumb() {
  return (
    <Breadcrumb>
      <BreadcrumbList>
        <BreadcrumbItem>
          <BreadcrumbLink href="/">Home</BreadcrumbLink>
        </BreadcrumbItem>
        <BreadcrumbSeparator />
        <BreadcrumbItem>
          <BreadcrumbLink href="/products">Products</BreadcrumbLink>
        </BreadcrumbItem>
        <BreadcrumbSeparator />
        <BreadcrumbItem>
          <BreadcrumbPage>Current Page</BreadcrumbPage>
        </BreadcrumbItem>
      </BreadcrumbList>
    </Breadcrumb>
  )
}
```

## Accessibility

- Uses proper semantic HTML with `nav` and `ol` elements
- Includes `aria-label` for screen readers
- Current page is marked with `aria-current="page"`
- Keyboard navigation support for all interactive elements

## Best Practices

1. **Keep It Simple**: Don't include too many levels in the breadcrumb
2. **Current Page**: Always mark the current page clearly
3. **Consistent Styling**: Use consistent styling across your application
4. **Mobile Consideration**: Consider truncating or collapsing on mobile devices
5. **Meaningful Labels**: Use clear, descriptive labels for each breadcrumb item
