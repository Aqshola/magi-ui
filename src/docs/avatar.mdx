# Avatar

An avatar component for displaying user profile pictures with customizable fallback text and MAGI-themed styling variants.

## Features

- **Image Support**: Display user profile pictures with automatic fallback
- **Size Variants**: Multiple size options from extra small to 2XL
- **MAGI Themes**: Specialized variants for different system roles and statuses
- **Accessibility**: Built on Radix UI primitives with proper ARIA support
- **Responsive Design**: Scales appropriately across different screen sizes
- **Customizable**: Extensive styling options and custom CSS support

## Size Reference

| Size | Dimensions | Use Case |
|------|------------|----------|
| `xs` | 24px | Small lists, compact layouts |
| `sm` | 32px | Navigation bars, inline mentions |
| `md` | 40px | Default size, general use |
| `lg` | 48px | Profile cards, prominent display |
| `xl` | 56px | Large profile sections |
| `2xl` | 64px | Hero sections, main profiles |

## Variant Reference

| Variant | Description | Use Case |
|---------|-------------|----------|
| `default` | Standard neutral styling | Regular users, general purpose |
| `magi` | MAGI system themed with gradient | Core system components, administrators |
| `system` | System-level styling | System processes, automated accounts |
| `warning` | Warning state styling | Alert conditions, attention needed |
| `danger` | Critical/error state styling | Critical alerts, error states |

## Accessibility

The Avatar component is built on Radix UI primitives and includes:

- **Image Loading**: Automatic fallback when images fail to load
- **Alt Text**: Proper alternative text support for screen readers
- **Semantic HTML**: Uses appropriate HTML elements for accessibility
- **Keyboard Navigation**: Supports keyboard interaction when interactive
- **ARIA Support**: Includes proper ARIA attributes from Radix UI

### Best Practices

1. **Always provide alt text** for avatar images
2. **Use meaningful fallback text** (usually user initials)
3. **Choose appropriate sizes** for the context
4. **Use variants consistently** across your application
5. **Consider loading states** for dynamic content

## Usage Guidelines

### When to Use

- **User profiles** and account displays
- **Team member lists** and directories
- **System status indicators** with personification
- **Comment sections** and user-generated content
- **Navigation elements** showing current user

### Best Practices

1. **Consistent Sizing**: Use consistent avatar sizes within the same context
2. **Meaningful Fallbacks**: Use initials or relevant icons as fallbacks
3. **Appropriate Variants**: Choose variants that match the user's role or status
4. **Loading States**: Handle image loading gracefully
5. **Accessibility**: Always provide proper alt text and fallback content

### Common Patterns

```tsx
// User profile with status
<Avatar variant="magi" size="lg">
  <AvatarImage src={user.avatar} alt={user.name} />
  <AvatarFallback variant="magi">{user.initials}</AvatarFallback>
</Avatar>

// System process indicator
<Avatar variant="system" size="sm">
  <AvatarFallback variant="system">DB</AvatarFallback>
</Avatar>

// Team member list
<div className="flex -space-x-2">
  {team.map(member => (
    <Avatar key={member.id} size="md" className="border-2 border-background">
      <AvatarImage src={member.avatar} alt={member.name} />
      <AvatarFallback>{member.initials}</AvatarFallback>
    </Avatar>
  ))}
</div>
```
