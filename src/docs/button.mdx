## Overview

The Button component is a fundamental interactive element in the MAGI-UI system. It provides consistent styling, behavior, and accessibility features across different variants and use cases.

## Features

- **Multiple Variants**: Default, destructive, outline, secondary, ghost, and link styles
- **Size Options**: Small, default, large, and icon sizes
- **Loading States**: Built-in loading spinner support
- **Icon Support**: Leading and trailing icon placement
- **Accessibility**: Full keyboard navigation and screen reader support
- **Polymorphic**: Can render as button, link, or custom elements

## API Reference

### Button Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "destructive" \| "outline" \| "secondary" \| "ghost" \| "link"` | `"default"` | Visual style variant |
| `size` | `"default" \| "sm" \| "lg" \| "icon"` | `"default"` | Size of the button |
| `asChild` | `boolean` | `false` | Render as child element (polymorphic) |
| `disabled` | `boolean` | `false` | Disable button interaction |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Button content |

### Variant Descriptions

- **default**: Primary button with solid background
- **destructive**: Red variant for dangerous actions
- **outline**: Transparent background with border
- **secondary**: Muted background for secondary actions
- **ghost**: Minimal styling, transparent background
- **link**: Styled as a text link

## Accessibility

- **Keyboard Navigation**: Full support for Tab, Enter, and Space keys
- **Screen Readers**: Proper button semantics and ARIA attributes
- **Focus Management**: Visible focus indicators and logical tab order
- **Disabled State**: Proper disabled semantics and visual feedback

## Best Practices

1. **Use semantic variants**: Choose variants that match the action's importance
2. **Include loading states**: Show feedback during async operations
3. **Provide clear labels**: Use descriptive text that explains the action
4. **Consider icon placement**: Use icons to enhance, not replace, text labels
5. **Test accessibility**: Ensure buttons work with keyboard and screen readers
