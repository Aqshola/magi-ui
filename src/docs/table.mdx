## Overview

The Table component provides a flexible and accessible way to display structured data. It uses semantic HTML table elements with enhanced styling and supports various patterns like sorting, selection, and responsive layouts.

## Features

- **Semantic HTML**: Built with proper table semantics for accessibility
- **Responsive Design**: Adapts to different screen sizes with horizontal scrolling
- **Flexible Layout**: Supports headers, footers, captions, and custom cell content
- **Styling Support**: Consistent with design system using Tailwind CSS
- **Accessibility**: Screen reader friendly with proper ARIA attributes
- **Customizable**: Easy to style and extend with additional functionality
- **Data Slots**: Uses data-slot attributes for enhanced styling hooks

<ComponentPreview
  title="Simple Table"
  description="Basic table with invoice data showing headers, body, and caption"

        invoice: "INV001",

        paymentMethod: "PayPal",
      },
      {
        invoice: "INV003",

    return (
      <Table>

## Accessibility

- **Semantic HTML**: Uses proper table elements (`table`, `thead`, `tbody`, `tfoot`, `tr`, `th`, `td`)
- **ARIA Support**: Automatic ARIA attributes for screen readers
- **Keyboard Navigation**: Tab navigation between interactive elements
- **Focus Management**: Proper focus indicators for interactive elements
- **Screen Reader**: Compatible with screen reader software

## Best Practices

1. **Use Semantic Structure**: Always use proper table elements and structure
2. **Provide Captions**: Use `TableCaption` to describe table content
3. **Clear Headers**: Use descriptive column headers with `TableHead`
4. **Consistent Alignment**: Align content appropriately (text left, numbers right)
5. **Loading States**: Show skeleton loaders during data fetching
6. **Empty States**: Provide helpful messages when no data is available
7. **Responsive Design**: Consider mobile layouts and horizontal scrolling
8. **Performance**: Implement pagination or virtualization for large datasets
