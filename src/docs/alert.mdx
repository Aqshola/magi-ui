## Overview

The Alert component provides a way to display important information, warnings, errors, or success messages to users. It supports different variants and can include icons, titles, and descriptions.

## Features

- **Multiple Variants**: Default, destructive, and custom styling options
- **Icon Support**: Built-in icon placement with customizable icons
- **Flexible Content**: Support for titles, descriptions, and custom content
- **Accessibility**: Full ARIA support and keyboard navigation
- **Responsive Design**: Adapts to different screen sizes and containers
- **Semantic HTML**: Uses appropriate semantic elements for better accessibility

## API Reference

### Alert Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "destructive"` | `"default"` | Alert variant style |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Alert content |

### AlertTitle Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Title content |

### AlertDescription Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Description content |

## Accessibility

- **Semantic HTML**: Uses appropriate ARIA roles and attributes
- **Screen Readers**: Proper announcement of alert content and severity
- **Keyboard Navigation**: Focusable when containing interactive elements
- **Color Contrast**: Meets WCAG guidelines for text contrast
- **Live Regions**: Automatically announced when dynamically added

## Best Practices

1. **Use appropriate variants**: Choose the right variant for the message type
2. **Include meaningful icons**: Use icons that reinforce the message context
3. **Keep content concise**: Write clear, actionable messages
4. **Provide context**: Include enough information for users to understand and act
5. **Consider placement**: Position alerts where users expect to see them
6. **Test accessibility**: Ensure alerts work with screen readers and keyboard navigation
