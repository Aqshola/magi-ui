## Overview

The Label component provides accessible labeling for form controls. It ensures proper semantic association between labels and their corresponding form elements, improving accessibility and user experience.

## Features

- **Semantic Association**: Proper `htmlFor` attribute linking to form controls
- **Accessibility**: Full screen reader support and keyboard navigation
- **Flexible Styling**: Customizable appearance with consistent typography
- **Form Integration**: Works seamlessly with all form components
- **Required Indicators**: Support for required field indicators

## API Reference

### Label Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `htmlFor` | `string` | - | ID of the associated form control |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Label content |

### HTML Attributes

The Label component accepts all standard HTML `<label>` attributes:

- `htmlFor` - Associates the label with a form control
- `onClick` - Click event handler
- `onMouseDown` - Mouse down event handler
- All other standard HTML label attributes

## Design Tokens

```css
/* Label base styles */
.label {
  font-size: 0.875rem;
  font-weight: 500;
  line-height: 1;
  color: hsl(var(--foreground));
  cursor: pointer;
}

.label:hover {
  color: hsl(var(--foreground) / 0.8);
}

.label[data-disabled] {
  cursor: not-allowed;
  opacity: 0.7;
}

.label[data-invalid] {
  color: hsl(var(--destructive));
}
```

## Accessibility

- **Semantic Association**: Uses `htmlFor` attribute to associate with form controls
- **Screen Readers**: Properly announced by assistive technologies
- **Keyboard Navigation**: Clicking label focuses associated form control
- **Focus Management**: Maintains proper focus relationships
- **ARIA Support**: Works with ARIA attributes on form controls

## Best Practices

1. **Always associate labels**: Use `htmlFor` to link labels with form controls
2. **Use descriptive text**: Write clear, concise label text
3. **Indicate required fields**: Use visual indicators for required fields
4. **Maintain consistency**: Use consistent label styling throughout forms
5. **Consider accessibility**: Ensure labels work with screen readers
6. **Test keyboard navigation**: Verify labels work with keyboard-only navigation
