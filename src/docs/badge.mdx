## Overview

The Badge component provides a compact way to display status information, categories, or metadata. It supports multiple variants and can be used to highlight important information or categorize content.

## Features

- **Multiple Variants**: Default, secondary, destructive, and outline styles
- **Flexible Content**: Support for text, numbers, and icons
- **Responsive Design**: Adapts to different screen sizes and containers
- **Semantic HTML**: Uses appropriate semantic elements
- **Customizable**: Easy to extend with custom styles and variants

## API Reference

### Badge Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"default" \| "secondary" \| "destructive" \| "outline"` | `"default"` | Badge variant style |
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Badge content |

### Variant Descriptions

- **default**: Primary badge style with solid background
- **secondary**: Muted badge style for less prominent information
- **destructive**: Red badge style for errors or critical states
- **outline**: Bordered badge style with transparent background

## Design Tokens

```css
/* Badge base styles */
.badge {
  display: inline-flex;
  align-items: center;
  border-radius: 9999px;
  padding: 0.125rem 0.625rem;
  font-size: 0.75rem;
  font-weight: 600;
  line-height: 1;
  transition: background-color 0.2s, border-color 0.2s, color 0.2s;
}

.badge-default {
  border: 1px solid transparent;
  background-color: hsl(var(--primary));
  color: hsl(var(--primary-foreground));
}

.badge-secondary {
  border: 1px solid transparent;
  background-color: hsl(var(--secondary));
  color: hsl(var(--secondary-foreground));
}

.badge-destructive {
  border: 1px solid transparent;
  background-color: hsl(var(--destructive));
  color: hsl(var(--destructive-foreground));
}

.badge-outline {
  border: 1px solid hsl(var(--border));
  color: hsl(var(--foreground));
}
```

## Accessibility

- **Semantic HTML**: Uses appropriate semantic elements
- **Screen Readers**: Properly announced by assistive technologies
- **Color Contrast**: Meets WCAG guidelines for text contrast
- **Focus States**: Focusable when containing interactive content

## Best Practices

1. **Use appropriate variants**: Choose variants that match the semantic meaning
2. **Keep content concise**: Use short, descriptive text
3. **Consider context**: Place badges where they provide meaningful information
4. **Maintain consistency**: Use consistent badge styles throughout your application
5. **Test accessibility**: Ensure badges are readable and accessible
6. **Avoid overuse**: Don't overwhelm interfaces with too many badges
