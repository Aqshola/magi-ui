## Overview

The Menubar component provides a traditional desktop-style application menu interface. It's perfect for applications that need persistent access to commands and actions, similar to the menu bars found in desktop applications.

## Features

- **Desktop-style Navigation**: Traditional application menubar interface
- **Keyboard Navigation**: Full keyboard support with arrow key navigation
- **Nested Submenus**: Support for multi-level menu hierarchies
- **Checkbox/Radio Items**: Interactive menu items with state
- **Keyboard Shortcuts**: Display shortcuts for quick access
- **Customizable**: Flexible styling and theming options
- **Accessibility**: Full ARIA support and screen reader compatibility
- **Portal Rendering**: Menus render in portals to avoid z-index issues

## API Reference

### Menubar

The root menubar container.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |

### MenubarMenu

Individual menu within the menubar.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |

### MenubarTrigger

Clickable trigger that opens the menu.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `disabled` | `boolean` | `false` | Whether the trigger is disabled |

### MenubarContent

Container for menu items.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `align` | `"start" \| "center" \| "end"` | `"start"` | Alignment relative to trigger |
| `sideOffset` | `number` | `4` | Distance from trigger |

### MenubarItem

Individual menu item.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `disabled` | `boolean` | `false` | Whether the item is disabled |
| `onSelect` | `() => void` | - | Callback when item is selected |

### MenubarCheckboxItem

Menu item with checkbox state.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `checked` | `boolean` | `false` | Whether the item is checked |
| `onCheckedChange` | `(checked: boolean) => void` | - | Callback when checked state changes |
| `className` | `string` | - | Additional CSS classes |

### MenubarRadioGroup

Container for radio menu items.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | Currently selected value |
| `onValueChange` | `(value: string) => void` | - | Callback when value changes |

### MenubarRadioItem

Radio menu item within a group.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | Value of this radio item |
| `className` | `string` | - | Additional CSS classes |

### MenubarSub

Container for submenu.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |

### MenubarSubTrigger

Trigger that opens a submenu.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `disabled` | `boolean` | `false` | Whether the trigger is disabled |

### MenubarSubContent

Container for submenu items.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |

### MenubarSeparator

Visual separator between menu items.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |

### MenubarShortcut

Keyboard shortcut display.

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |

```tsx
          </MenubarItem>
```

```tsx
      </MenubarMenu>
    </Menubar>
  )
}
```

## Accessibility

- Full keyboard navigation with arrow keys
- Proper ARIA attributes for screen readers
- Focus management between menu items
- Support for keyboard shortcuts

## Best Practices

1. **Content**: Use clear, descriptive labels for menu items
2. **Performance**: Avoid complex content in menu items
3. **User Experience**: Group related items with separators
