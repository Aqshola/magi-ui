## Overview

The Radio Group component allows users to select exactly one option from a set of mutually exclusive choices. Built on top of Radix UI's RadioGroup primitive, it provides excellent accessibility and keyboard navigation support.

## Features

- **Single Selection**: Only one option can be selected at a time
- **Keyboard Navigation**: Full keyboard accessibility with arrow keys
- **Flexible Layout**: Support for various layouts and orientations
- **Rich Content**: Support for icons, descriptions, and complex layouts
- **Controlled/Uncontrolled**: Both controlled and uncontrolled usage patterns
- **Disabled State**: Support for disabled options and entire groups
- **Accessible**: Built-in ARIA attributes and screen reader support
- **Customizable**: Easy to style and extend with custom designs

## API Reference

### RadioGroup

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | `undefined` | Controlled value |
| `defaultValue` | `string` | `undefined` | Default value for uncontrolled usage |
| `onValueChange` | `(value: string) => void` | `undefined` | Callback when value changes |
| `disabled` | `boolean` | `false` | Whether the entire group is disabled |
| `required` | `boolean` | `false` | Whether selection is required |
| `name` | `string` | `undefined` | Name attribute for form submission |
| `orientation` | `"horizontal" \| "vertical"` | `"vertical"` | Layout orientation |
| `dir` | `"ltr" \| "rtl"` | `undefined` | Text direction |
| `loop` | `boolean` | `true` | Whether keyboard navigation should loop |
| `className` | `string` | `undefined` | Additional CSS classes |

### RadioGroupItem

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | `undefined` | The value for this radio item |
| `disabled` | `boolean` | `false` | Whether this item is disabled |
| `required` | `boolean` | `false` | Whether this item is required |
| `id` | `string` | `undefined` | ID for label association |
| `className` | `string` | `undefined` | Additional CSS classes |

Both components accept all standard HTML attributes for their respective elements.

## Accessibility

The Radio Group component follows WAI-ARIA guidelines:

- **Keyboard Navigation**: Full keyboard support with arrow keys
- **Screen Reader**: Proper ARIA attributes and announcements
- **Focus Management**: Clear focus indicators and logical tab order
- **Label Association**: Proper label-input relationships

### Keyboard Shortcuts

| Key | Action |
|-----|--------|
| `↓` `→` | Move to next option |
| `↑` `←` | Move to previous option |
| `Space` | Select focused option |
| `Tab` | Move focus to next element |
| `Shift + Tab` | Move focus to previous element |

## Best Practices

### When to Use Radio Groups

- **Single selection**: When users need to select exactly one option
- **Mutually exclusive choices**: Payment methods, shipping options, settings
- **Visible options**: When all available options should be visible at once
- **Immediate feedback**: When selection changes need immediate visual feedback
- **2-7 options**: Optimal range for radio group usability

### When to Avoid Radio Groups

- **Multiple selection**: Use checkboxes instead
- **Binary choices**: Use toggle or switch instead
- **Many options**: Use select dropdown instead
- **Immediate actions**: Use buttons instead
- **Obvious defaults**: When the default is rarely changed

### Design Guidelines

- **Clear labels**: Use descriptive, concise labels for each option
- **Logical ordering**: Order options alphabetically, by popularity, or importance
- **Consistent formatting**: Maintain consistent styling across all options
- **Adequate spacing**: Ensure sufficient space between options for easy selection
- **Visual hierarchy**: Use visual cues to emphasize important options

### Performance Considerations

- **Controlled vs Uncontrolled**: Use uncontrolled for simple forms, controlled for complex state management
- **Large lists**: Consider virtualization for very large option lists
- **Form validation**: Validate selection on form submission rather than on every change

```tsx
// ✅ Good: Uncontrolled for simple cases
<RadioGroup defaultValue="option1" name="simple-choice">
  {/* Options */}
</RadioGroup>

// ✅ Good: Controlled for complex state
const [selection, setSelection] = useState("")
<RadioGroup value={selection} onValueChange={setSelection}>
  {/* Options */}
</RadioGroup>

// ✅ Good: Proper form integration
<form onSubmit={handleSubmit}>
  <RadioGroup name="user-choice" required>
    {/* Options */}
  </RadioGroup>
</form>
```
