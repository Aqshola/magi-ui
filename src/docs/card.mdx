## Overview

The Card component provides a structured container for organizing related content. It follows a modular composition pattern with separate components for header, content, and footer sections.

## Features

- **Modular Composition**: Separate components for different card sections
- **Flexible Layout**: Adaptable to various content types and layouts
- **Consistent Spacing**: Predefined padding and spacing patterns
- **Semantic Structure**: Proper HTML semantics for accessibility
- **Responsive Design**: Works seamlessly across different screen sizes

## API Reference

### Card Components

| Component | Description |
|-----------|-------------|
| `Card` | Root container with border and background |
| `CardHeader` | Header section with consistent padding |
| `CardTitle` | Primary heading with appropriate typography |
| `CardDescription` | Secondary text with muted styling |
| `CardContent` | Main content area with flexible layout |
| `CardFooter` | Footer section for actions or metadata |

### Card Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `className` | `string` | - | Additional CSS classes |
| `children` | `ReactNode` | - | Card content |

All card components accept standard HTML div props and can be customized with additional classes.

## Design Tokens

```css
/* Card base styles */
.card {
  border-radius: 0.5rem;
  border: 1px solid hsl(var(--border));
  background-color: hsl(var(--card));
  color: hsl(var(--card-foreground));
  box-shadow: 0 1px 3px 0 rgb(0 0 0 / 0.1);
}

/* Card sections */
.card-header { padding: 1.5rem 1.5rem 0; }
.card-content { padding: 1.5rem; }
.card-footer { 
  padding: 0 1.5rem 1.5rem;
  display: flex;
  align-items: center;
}
```

## Accessibility

- **Semantic Structure**: Uses proper heading hierarchy and landmarks
- **Keyboard Navigation**: Focusable elements maintain logical tab order
- **Screen Readers**: Clear content structure and meaningful headings
- **Color Contrast**: Meets WCAG guidelines for text and background contrast

## Best Practices

1. **Use semantic headings**: Maintain proper heading hierarchy with CardTitle
2. **Group related content**: Keep logically related information together
3. **Provide clear actions**: Use CardFooter for primary actions
4. **Consider responsive design**: Test cards across different screen sizes
5. **Maintain consistency**: Use consistent spacing and typography patterns
